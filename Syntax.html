Case Sensitivity in Java

In Java, how you use uppercase and lowercase letters matters a lot! Here are some important points to remember:

  1. Class Names: Start with an uppercase letter. For example, HelloWorld is correct, but helloworld is not.
  2. Method Names: Always use lowercase for method names. The main method is always written as main, not Main.
  3. Keywords: All keywords and access modifiers, like public, class, and static must be in lowercase.
  4. Predefined Classes: Built-in classes, like System, must keep their original case. Writing it as system will cause an error.

Java Code Explanation

public class HelloWorld {
  • public: This is an access modifier that allows the class to be accessible from other classes. It is always written in lowercase.
  • class: This keyword is used to declare a class. It must be in lowercase.
  • HelloWorld: This is the name of the class. By convention, class names start with an uppercase letter. This specific name must match the filename (HelloWorld.java) exactly (including case) to compile successfully. 
public static void main(String [] args) {
  • public: Again, this is an access modifier indicating that the method is accessible from anywhere.
  • static: This keyword indicates that the method belongs to the class rather than instances of the class. It allows the main method to be called without creating an instance of the class.
  • void: This specifies that the method does not return a value.
  • main: This is the name of the method. It must be written in lowercase. This is the entry point of the Java application, and it must be named exactly main (case-sensitive).
  • String [] args:
    • String: This indicates that args is an array of String objects. The type name String must start with an uppercase letter, as it is a predefined class in Java.
    • []: Indicates that args is an array. The brackets come immediately after the type.
